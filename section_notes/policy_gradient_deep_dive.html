Policy Gradient
from First Principles

A line-by-line derivation from the definition of an MDP through REINFORCE, variance reduction, Actor-Critic, and PPO — built so you can teach every step.

01

MDP & First Principles

Before we can optimize anything, we need to be precise about what world we are operating in. That world is a Markov Decision Process.

1.1 The Markov Decision Process

A Markov Decision Process is a mathematical framework for sequential decision-making under uncertainty. It is defined by a tuple \(\mathcal{M} = (\mathcal{S}, \mathcal{A}, p, r, \gamma)\):

\(\mathcal{S}\)
State space
The set of all possible situations the agent can find itself in. Could be discrete (chess positions) or continuous (robot joint angles).
\(\mathcal{A}\)
Action space
The set of all actions the agent can choose. Again, discrete (up/down/left/right) or continuous (torque values).
\(p(s'|s,a)\)
Transition dynamics
The probability of landing in state \(s'\) when taking action \(a\) in state \(s\). This encodes the physics of the world.
\(r(s,a)\)
Reward function
The scalar feedback signal: how desirable is taking action \(a\) in state \(s\)? This encodes what we want the agent to do.
\(\gamma \in [0,1)\)
Discount factor
How much the agent values future rewards relative to immediate ones. \(\gamma=0\): only care about now. \(\gamma \to 1\): plan infinitely far ahead.
\(p(s_1)\)
Initial state distribution
Probability distribution over starting states. The agent has no control over this — it's part of the environment.

The crucial word is Markov: the probability of the next state depends only on the current state and action, not on any history. This is written:

Markov Property
\[ p(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, \ldots, s_1, a_1) = p(s_{t+1} | s_t, a_t) \]
Intuition
The state must be a sufficient statistic of the past. For chess, the board position is the entire relevant history — you don't need to know how you arrived there. For a robot arm, joint positions and velocities form the state because Newton's laws only depend on current configuration, not history. If your state representation satisfies the Markov property, the MDP framework is exact. If not, it is an approximation.

1.2 What is a Policy?

The agent's behavior is described by a policy \(\pi\). A policy is a mapping from states to actions — it answers the question: given where I am, what should I do?

Policies can be deterministic: \(\pi: \mathcal{S} \to \mathcal{A}\), meaning a single action is selected with certainty. But for our purposes we work with stochastic policies:

Stochastic Policy
\[ \pi_\theta(a | s) = \Pr(A_t = a \mid S_t = s;\, \theta) \]

Here \(\theta\) are the parameters of the policy — think of the weights of a neural network that takes state \(s\) as input and outputs a probability distribution over actions. We want to find the best \(\theta\).

Why stochastic?
A deterministic policy is a special case. But stochastic policies have advantages: (1) they naturally model exploration — the agent might occasionally try suboptimal actions and discover they're actually better; (2) in partially observable or adversarial environments, mixing actions prevents the opponent from predicting you; (3) the gradient of a stochastic policy with respect to \(\theta\) is well-defined and non-zero everywhere, which is essential for gradient-based optimization.

1.3 Return and Expectation

A single interaction produces a trajectory \(\tau\): a sequence of states, actions, and rewards over time.

Trajectory
\[ \tau = (s_1, a_1, r_1, s_2, a_2, r_2, \ldots, s_T, a_T, r_T) \]

The return \(G(\tau)\) is the total accumulated (discounted) reward along a trajectory:

Discounted Return
\[ G(\tau) = \sum_{t=1}^{T} \gamma^{t-1} r(s_t, a_t) \]

The discount \(\gamma^{t-1}\) makes rewards further in the future count for less. This is both mathematically convenient (it ensures the sum is finite in infinite-horizon problems) and economically natural (a reward now is generally worth more than the same reward later, because of uncertainty about what happens between now and then).

Since trajectories are random — the initial state is drawn from \(p(s_1)\), actions are sampled from \(\pi_\theta(a|s)\), and next states from \(p(s'|s,a)\) — the return \(G(\tau)\) is a random variable. We care about its expected value:

Expected Return — the thing we want to maximize
\[ J(\theta) = \mathbb{E}_{\tau \sim p_\theta(\tau)}\bigl[G(\tau)\bigr] \]

The expectation \(\mathbb{E}\) is taken over all possible trajectories, weighted by how likely each trajectory is under our policy \(\pi_\theta\). This is the central object of all policy gradient methods.

02

Trajectory Probability

To compute the expectation in \(J(\theta)\), we need an explicit expression for how likely any given trajectory is. Let us build it from scratch.

2.1 Factoring the Joint Probability

A trajectory \(\tau = (s_1, a_1, s_2, a_2, \ldots, s_T, a_T)\) occurs through a chain of events. Using the chain rule of probability and the Markov property:

\(p_\theta(\tau) = p(s_1, a_1, s_2, a_2, \ldots, s_T, a_T)\)
Start: joint probability of the entire sequence
\(= p(s_1) \cdot \pi_\theta(a_1|s_1) \cdot p(s_2|s_1,a_1) \cdots\)
Chain rule: factor each event given the previous ones. The Markov property makes \(p(s_{t+1})\) depend only on \((s_t, a_t)\).
\(= p(s_1) \prod_{t=1}^{T} \pi_\theta(a_t|s_t)\, p(s_{t+1}|s_t, a_t)\)
Compact form: the product structure reflects that each timestep is conditionally independent given the previous one.
Trajectory Distribution (key formula)
\[ p_\theta(\tau) = p(s_1) \prod_{t=1}^{T} \pi_\theta(a_t|s_t)\, p(s_{t+1}|s_t, a_t) \]

This decomposition has a critical structure: the factors split into three groups.

\(p(s_1)\)
Initial state distribution — entirely outside our control. We cannot optimize this with \(\theta\).
\(\pi_\theta(a_t|s_t)\)
Policy's action probabilities — the only factor that depends on \(\theta\). This is where our learning happens.
\(p(s_{t+1}|s_t,a_t)\)
Environment dynamics — also outside our control. We typically don't know this distribution explicitly (it's the physics/simulator).
The Key Insight
Only the policy terms \(\pi_\theta(a_t|s_t)\) depend on \(\theta\). The initial distribution \(p(s_1)\) and dynamics \(p(s_{t+1}|s_t,a_t)\) are properties of the environment, not the agent. This observation is what enables the entire derivation — and it's why the log-derivative trick will eliminate the terms we don't know.
03

The Objective Function J(θ)

3.1 What Are We Maximizing?

We can now write the full objective in explicit integral form:

Objective — integral form
\[ J(\theta) = \mathbb{E}_{\tau \sim p_\theta(\tau)}[r(\tau)] = \int p_\theta(\tau)\, r(\tau)\, d\tau \]

where we use shorthand \(r(\tau) = \sum_{t=1}^T r(s_t, a_t)\) for the undiscounted return along trajectory \(\tau\). (In practice one adds \(\gamma\), but we follow the convention in the notes where \(\gamma\) is absorbed or treated as \(\approx 1\) for derivation clarity.)

3.2 Why This is the Right Objective

You might ask: why expectation? Why not maximize the return of the single best trajectory?

The answer is that we want a policy — a consistent decision rule — not a lucky sequence. Maximizing expected return forces the policy to be good on average across all possible starting states and all possible random outcomes in the environment. A policy that gets lucky once but fails in general is not what we want.

Furthermore, we can equivalently write:

Alternative form — sum over timesteps
\[ \theta^* = \arg\max_\theta \sum_{t=1}^T \mathbb{E}_{(s_t, a_t) \sim p_\theta(s_t, a_t)}\bigl[r(s_t, a_t)\bigr] \]

This form shows that maximizing total expected return is equivalent to maximizing expected reward at every timestep simultaneously. The marginal distributions \(p_\theta(s_t, a_t)\) depend on \(\theta\) because the policy determines which states are visited.

3.3 Monte Carlo Approximation

The integral \(\int p_\theta(\tau) r(\tau) d\tau\) is intractable in general — the space of all trajectories is astronomically large. In practice, we estimate it by sampling \(N\) trajectories:

Monte Carlo Estimator of J(θ)
\[ J(\theta) \approx \frac{1}{N} \sum_{i=1}^N \sum_{t=1}^T r(s_{i,t},\, a_{i,t}) \]

This is an unbiased estimator: its expected value is exactly \(J(\theta)\). But it can have high variance when individual trajectories vary a lot in their returns — a problem we'll address later.

04

The Log-Derivative Trick

We want to maximize \(J(\theta)\), which means we need its gradient \(\nabla_\theta J(\theta)\). Let us compute it.

4.1 The Problem

The naive approach: differentiate inside the integral.

Starting point
\[ \nabla_\theta J(\theta) = \nabla_\theta \int p_\theta(\tau)\, r(\tau)\, d\tau = \int \nabla_\theta p_\theta(\tau)\, r(\tau)\, d\tau \]

(We are allowed to move the gradient inside the integral under mild regularity conditions, which we assume hold.)

Now we have \(\nabla_\theta p_\theta(\tau)\). The trouble: we cannot sample from \(\nabla_\theta p_\theta(\tau)\) — it is not a probability distribution (it can be negative!). We need to convert this into an expectation so we can estimate it with samples.

4.2 The Trick: Log-Derivative Identity

There is a beautiful calculus identity that saves us. For any differentiable function \(f(\theta) > 0\):

Log-Derivative Identity
\[ \nabla_\theta \log f(\theta) = \frac{\nabla_\theta f(\theta)}{f(\theta)} \quad \Longleftrightarrow \quad \nabla_\theta f(\theta) = f(\theta)\, \nabla_\theta \log f(\theta) \]

This is nothing but the chain rule: \(\frac{d}{d\theta} \log f = \frac{1}{f} \cdot f'\). It is trivial as calculus, but extraordinarily useful here.

4.3 Applying the Trick

We apply the identity \(\nabla_\theta p_\theta(\tau) = p_\theta(\tau) \cdot \nabla_\theta \log p_\theta(\tau)\) inside our integral. Let's go step by step:

\(\displaystyle \int \nabla_\theta p_\theta(\tau)\, r(\tau)\, d\tau\)
Where we are: gradient of p, not sampleable.
\(\displaystyle = \int p_\theta(\tau) \cdot \frac{\nabla_\theta p_\theta(\tau)}{p_\theta(\tau)} \cdot r(\tau)\, d\tau\)
Multiply and divide by \(p_\theta(\tau)\). Legal because \(p_\theta(\tau) > 0\) everywhere (under continuous distributions).
\(\displaystyle = \int p_\theta(\tau) \cdot \nabla_\theta \log p_\theta(\tau) \cdot r(\tau)\, d\tau\)
Apply log-derivative identity: \(\frac{\nabla_\theta p_\theta(\tau)}{p_\theta(\tau)} = \nabla_\theta \log p_\theta(\tau)\)
\(\displaystyle = \mathbb{E}_{\tau \sim p_\theta(\tau)}\bigl[\nabla_\theta \log p_\theta(\tau) \cdot r(\tau)\bigr]\)
Recognize as expectation: \(\int p_\theta(\tau)(\cdot)\,d\tau = \mathbb{E}_{p_\theta}[\cdot]\). Now sampleable!
Policy Gradient (general form)
\[ \nabla_\theta J(\theta) = \mathbb{E}_{\tau \sim p_\theta(\tau)}\bigl[\nabla_\theta \log p_\theta(\tau) \cdot r(\tau)\bigr] \]

4.4 Why This Trick Is Profound

Before the trick, we had a quantity we could not estimate with samples. After the trick, we have an expectation under \(p_\theta\) — which we estimate simply by running the policy and averaging. Specifically:

What we gained

Before: \(\int \nabla_\theta p_\theta(\tau) r(\tau) d\tau\) — not an expectation under \(p_\theta\). Cannot be estimated by sampling trajectories.

After: \(\mathbb{E}_{p_\theta}[\nabla_\theta \log p_\theta(\tau) \cdot r(\tau)]\) — is an expectation under \(p_\theta\). Estimate by: sample trajectories from \(\pi_\theta\), compute \(\nabla_\theta \log p_\theta(\tau) \cdot r(\tau)\) for each, average them. Done.

This is also known as the likelihood ratio estimator or the score function estimator. The quantity \(\nabla_\theta \log p_\theta(\tau)\) is the score function of the trajectory distribution — a standard object in statistics.

05

Simplifying \(\nabla_\theta \log p_\theta(\tau)\)

We now expand \(\nabla_\theta \log p_\theta(\tau)\) using the explicit form of \(p_\theta(\tau)\) derived in Chapter 2. This is where the unknown environment terms vanish.

5.1 Taking the Log

Recall \(p_\theta(\tau) = p(s_1) \prod_{t=1}^{T} \pi_\theta(a_t|s_t)\, p(s_{t+1}|s_t, a_t)\). The logarithm converts products to sums:

Expanding the log
\[ \log p_\theta(\tau) = \log p(s_1) + \sum_{t=1}^{T} \log \pi_\theta(a_t|s_t) + \sum_{t=1}^{T} \log p(s_{t+1}|s_t, a_t) \]

5.2 Taking the Gradient w.r.t. θ

Now we differentiate with respect to \(\theta\). This is the beautiful step:

\(\nabla_\theta \log p_\theta(\tau) = \nabla_\theta \log p(s_1) + \nabla_\theta \sum_t \log \pi_\theta(a_t|s_t) + \nabla_\theta \sum_t \log p(s_{t+1}|s_t,a_t)\)
Linearity of gradient: gradient of a sum is the sum of gradients.
\(= \;0\; + \;\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t)\; + \;0\)
Key cancellation: \(p(s_1)\) and \(p(s_{t+1}|s_t,a_t)\) do not depend on \(\theta\), so their gradients are zero!
\(= \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t)\)
Result: only the policy terms survive. The environment is gone entirely.
Score of the trajectory — environment-free
\[ \nabla_\theta \log p_\theta(\tau) = \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t) \]
Why this is remarkable
We started with the trajectory distribution \(p_\theta(\tau)\), which includes the unknown dynamics \(p(s_{t+1}|s_t,a_t)\) and the unknown initial distribution \(p(s_1)\). After taking the log-gradient, both unknown terms vanish. Only the policy terms remain — and crucially, these are functions of \(\theta\) that we can compute directly! This is why model-free policy gradient methods can work without knowing the dynamics of the environment.

5.3 Intuition for the Score Function

What does \(\nabla_\theta \log \pi_\theta(a_t|s_t)\) mean intuitively? It is the direction in parameter space that increases the probability of action \(a_t\) in state \(s_t\). Think of it as a "voting signal": this quantity points in the direction "make action \(a_t\) more likely when in state \(s_t\)."

The policy gradient will take this score and weight it by the return \(r(\tau)\). So the final rule is: make an action more likely if the trajectory that contained it had high return; make it less likely if the return was low.

06

The REINFORCE Gradient

6.1 Plugging In

We now substitute our simplified score function back into the general policy gradient formula from Chapter 4:

Policy Gradient Theorem (REINFORCE form)
\[ \nabla_\theta J(\theta) = \mathbb{E}_{\tau \sim p_\theta(\tau)}\!\left[\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t)\right)\!\left(\sum_{t=1}^{T} r(s_t, a_t)\right)\right] \]

Let us decode each factor:

\(\sum_t \nabla_\theta \log \pi_\theta(a_t|s_t)\)
Total score of the trajectory. This is the direction to move \(\theta\) to make this entire sequence of actions more probable. Each term tells you: "shift \(\theta\) to make action \(a_t\) more likely in state \(s_t\)."
\(\sum_t r(s_t, a_t)\)
Total return of the trajectory. This acts as a scalar weight. If the trajectory was good (positive total reward), the score is added. If bad, it is subtracted. This is reinforcement learning in its purest form.

6.2 Monte Carlo Estimator

The expectation is estimated by collecting \(N\) trajectories:

Practical Gradient Estimator
\[ \nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N} \left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_{i,t}|s_{i,t})\right)\!\left(\sum_{t=1}^{T} r(s_{i,t}, a_{i,t})\right) \]

Subscript \(i\) indexes over trajectories, subscript \(t\) indexes over time within a trajectory. For trajectory \(i\), we compute the total score and total return, multiply them, and average across all \(N\) trajectories.

6.3 The Connection to Supervised Learning

Analogy with supervised learning
In supervised learning, we maximize log-likelihood: \(\nabla_\theta \sum_i \log p_\theta(y_i|x_i)\). The policy gradient looks almost the same: \(\nabla_\theta \sum_i \log \pi_\theta(a_i|s_i) \cdot R_i\). The difference is the weight \(R_i\): instead of treating every action equally (as supervised learning does), policy gradient weights each action by how good the trajectory was. Good trajectories get reinforced; bad ones get suppressed. This is precisely the "trial and error" nature of reinforcement learning.
07

The REINFORCE Algorithm

Algorithm: REINFORCE (Williams, 1992)
1.
Initialize policy parameters \(\theta\) randomly.
2.
Sample \(N\) trajectories \(\{\tau_i\}_{i=1}^N\) by running \(\pi_\theta\) in the environment. Each trajectory: start in \(s_1 \sim p(s_1)\), then repeatedly sample \(a_t \sim \pi_\theta(\cdot|s_t)\), observe \(s_{t+1} \sim p(\cdot|s_t, a_t)\), receive \(r_t = r(s_t, a_t)\), until episode ends.
3.
Estimate the gradient: \[ \widehat{\nabla_\theta J} = \frac{1}{N}\sum_{i=1}^N \left(\sum_t \nabla_\theta \log \pi_\theta(a_{i,t}|s_{i,t})\right)\!\left(\sum_t r(s_{i,t}, a_{i,t})\right) \]
4.
Update parameters via gradient ascent: \[ \theta \leftarrow \theta + \alpha\, \widehat{\nabla_\theta J} \]
5.
Repeat from step 2 until convergence.

7.1 An Important Extension: Partial Observability

Notice that throughout the derivation, we only required the Markov property to factor the trajectory probability. But after taking the log and gradient, the result is simply:

\[ \nabla_\theta J(\theta) = \mathbb{E}\!\left[\sum_t \nabla_\theta \log \pi_\theta(a_t|o_t) \cdot R\right] \]

where \(o_t\) is the observation (possibly partial). For partially observable MDPs (POMDPs), simply replace \(s_t\) with \(o_t\). The algorithm works identically — without requiring the environment to be Markovian! This is because we never used the Markov property in the gradient computation itself; only in writing down \(p_\theta(\tau)\), and those terms cancelled out anyway.

REINFORCE works beyond MDPs
This is a remarkable result: the REINFORCE gradient estimator is valid even when the problem is not Markovian. The agent might observe only part of the state (a camera image instead of full robot state), and the gradient formula remains correct — as long as the policy maps observations to action probabilities.
08

The High Variance Problem

8.1 Where Variance Comes From

REINFORCE is an unbiased estimator of the policy gradient — its expected value is exactly \(\nabla_\theta J(\theta)\). But it can have catastrophically high variance. Let us understand why.

Consider two trajectories in a robot navigation task. Both took the same action in state \(s_5\). Trajectory A ended successfully (total return = +100). Trajectory B failed (total return = −10). REINFORCE multiplies the score of every action by the total return — so action \(a_5\) gets gradient weight +100 in trajectory A and −10 in trajectory B, even though the action at step 5 may have had nothing to do with what happened at step 50.

Root Cause of High Variance
The return \(\sum_t r(s_t, a_t)\) includes rewards from all timesteps — including those that happened before action \(a_t\) was taken, and those far in the future that are determined by randomness in the environment, not by the quality of \(a_t\). Each additional random reward term added to the sum increases the variance of the estimator. For a trajectory of length \(T\), we are summing \(T\) random variables, each with its own variance.

8.2 Mathematical Analysis of Variance

The variance of the REINFORCE gradient estimator grows with:

Trajectory length T
Each additional timestep adds another random variable to the return sum. Variance scales roughly as \(O(T)\) for uncorrelated rewards.
Reward magnitude
Larger rewards mean larger fluctuations in the return, directly inflating gradient variance.
Environment stochasticity
Random transitions cause the same (state, action) to lead to very different future rewards across rollouts.
Sample size N
Variance of the Monte Carlo estimate decreases as \(1/N\). Reducing variance requires \(N^2\times\) as many samples — prohibitively expensive.

High variance means: the gradient estimate points in a very different direction each time you compute it. Gradient ascent with such noisy estimates is slow and unstable — it may take thousands of iterations to converge on a problem where a lower-variance method would converge in hundreds.

The practical question: how do we reduce variance while keeping the estimator unbiased? Two complementary techniques address this directly.

09

Causality & Reward-to-Go

9.1 The Principle of Causality

There is a fundamental temporal structure in MDPs: causes precede effects. The action taken at time \(t\) cannot affect rewards that occurred at times \(t' < t\) — those are already in the past.

Yet REINFORCE multiplies the score at time \(t\) by the full return \(\sum_{t'=1}^T r_{t'}\), which includes past rewards. These past rewards add noise without adding any useful gradient signal.

Formal Justification
We can show that for \(t' < t\), the contribution of \(r_{t'}\) to the gradient is zero in expectation: \[\mathbb{E}\left[\nabla_\theta \log \pi_\theta(a_t|s_t) \cdot r(s_{t'}, a_{t'})\right] = 0 \quad \text{for } t' < t\] This follows from the conditional independence structure of the MDP: the score function \(\nabla_\theta \log \pi_\theta(a_t|s_t)\) is mean-zero given the past (a property of score functions), and past rewards \(r_{t'}\) are independent of future policy decisions. The proof involves careful conditioning on the information available at time \(t\).

9.2 Reward-to-Go

Since past rewards contribute zero in expectation but add variance in practice, we simply drop them:

Step 1 — Rewrite as double sum
\[ \nabla_\theta J \approx \frac{1}{N}\sum_i \sum_{t=1}^T \nabla_\theta \log \pi_\theta(a_{i,t}|s_{i,t}) \underbrace{\left(\sum_{t'=1}^T r(s_{i,t'}, a_{i,t'})\right)}_{\text{full return (includes past)}} \]
Step 2 — Apply causality: drop past rewards
\[ \nabla_\theta J \approx \frac{1}{N}\sum_i \sum_{t=1}^T \nabla_\theta \log \pi_\theta(a_{i,t}|s_{i,t}) \underbrace{\left(\sum_{t'=t}^T r(s_{i,t'}, a_{i,t'})\right)}_{\hat{Q}_{i,t} \text{ — reward-to-go}} \]

The quantity \(\hat{Q}_{i,t} = \sum_{t'=t}^T r(s_{i,t'}, a_{i,t'})\) is called the reward-to-go (or empirical Q-value). It is the total return from timestep \(t\) onward. For a trajectory of length \(T\), this sum averages roughly \(T/2\) fewer terms than the full return — a substantial variance reduction, especially for long episodes.

\(\hat{Q}_{i,t}\)
Reward-to-go: empirical estimate of "how much total reward will I receive from this point forward?" This is a sample estimate of the true Q-function \(Q^\pi(s_t, a_t)\). It answers: "given that I took action \(a_t\) in state \(s_t\), what was the eventual total reward?"
10

Baselines & the Advantage Function

10.1 The Baseline Trick

We can further reduce variance by subtracting a baseline \(b(s_t)\) from the reward-to-go, as long as the baseline does not depend on the action \(a_t\).

Proof that baseline doesn't introduce bias
We need to show \(\mathbb{E}[\nabla_\theta \log \pi_\theta(a_t|s_t) \cdot b(s_t)] = 0\). Expanding: \[\mathbb{E}_{a_t \sim \pi_\theta(\cdot|s_t)}\!\left[\nabla_\theta \log \pi_\theta(a_t|s_t) \cdot b(s_t)\right] = b(s_t) \underbrace{\mathbb{E}_{a_t}\!\left[\nabla_\theta \log \pi_\theta(a_t|s_t)\right]}_{= 0}\] The expectation of the score function is always zero: \(\mathbb{E}_a[\nabla_\theta \log \pi_\theta(a|s)] = \nabla_\theta \int \pi_\theta(a|s)\,da = \nabla_\theta 1 = 0\). Therefore, subtracting any function \(b(s_t)\) that doesn't depend on \(a_t\) leaves the gradient unbiased.

10.2 Optimal Baseline

Not all baselines reduce variance equally. The optimal baseline (minimizing gradient variance) can be shown to be approximately equal to the value function:

Value function as baseline
\[ b(s_t) = V^\pi(s_t) = \mathbb{E}_{\tau \sim \pi_\theta, \tau \ni s_t}\!\left[\sum_{t'=t}^T r(s_{t'}, a_{t'})\right] \]

The value function \(V^\pi(s_t)\) is the expected reward-to-go from state \(s_t\) when following policy \(\pi\). It answers: "on average, how much reward do I get from state \(s_t\) onward?" This is the perfect baseline: it removes the average part of the return, leaving only the deviation from average — which is the genuinely informative signal.

10.3 The Advantage Function

With this baseline, the weight on each gradient term becomes:

Advantage Function
\[ A^\pi(s_t, a_t) = Q^\pi(s_t, a_t) - V^\pi(s_t) \]
\(Q^\pi(s_t, a_t)\)
Q-function: expected return when taking action \(a_t\) in state \(s_t\), then following \(\pi\). "How good is this specific action from here?"
\(V^\pi(s_t)\)
Value function: expected return from state \(s_t\) under \(\pi\), averaged over all actions. "How good is this state on average?"
\(A^\pi(s_t, a_t)\)
Advantage: how much better is action \(a_t\) compared to the average action in state \(s_t\)? Positive advantage means "this action was better than average, reinforce it." Negative advantage means "this action was worse than average, suppress it."

The final policy gradient with the advantage function:

Policy Gradient with Advantage (optimal form)
\[ \nabla_\theta J(\theta) \approx \frac{1}{N}\sum_i \sum_{t=1}^T \nabla_\theta \log \pi_\theta(a_{i,t}|s_{i,t}) \cdot A_{i,t} \]

In practice, \(A_{i,t} = \hat{Q}_{i,t} - V_\omega(s_{i,t})\), where \(\hat{Q}_{i,t}\) is the empirical reward-to-go and \(V_\omega\) is a learned value function (the critic). The advantage captures relative quality: not just "was this trajectory good?" but "was this specific action, in this specific state, better than what we'd normally expect?"

Intuition for variance reduction
The total return \(\hat{Q}_{i,t}\) fluctuates wildly because it includes large, random variations in how good the current state is — even if the action was perfectly chosen. The value function \(V^\pi(s_t)\) subtracts this "state quality" component. What remains — the advantage — is driven primarily by the quality of the action choice, not by the random goodness of the state. This is a much lower-variance signal, because we have factored out the dominant source of randomness.
11

Off-Policy Learning via Importance Sampling

REINFORCE is on-policy: it requires fresh trajectories from the current policy \(\pi_\theta\) for each gradient update. After updating \(\theta\), all previously collected data becomes invalid. This is sample-inefficient.

11.1 Importance Sampling Setup

Suppose we have trajectories collected from a reference policy \(\bar{\pi}\) (possibly an older version of \(\pi_\theta\)). We want to evaluate \(J(\theta)\) using these old trajectories. We use the importance sampling identity:

\(\displaystyle J(\theta) = \int p_\theta(\tau)\, r(\tau)\, d\tau\)
Original objective
\(\displaystyle = \int \bar{p}(\tau)\, \frac{p_\theta(\tau)}{\bar{p}(\tau)}\, r(\tau)\, d\tau\)
Multiply and divide by \(\bar{p}(\tau)\): the reference distribution.
\(\displaystyle = \mathbb{E}_{\tau \sim \bar{p}(\tau)}\!\left[\frac{p_\theta(\tau)}{\bar{p}(\tau)}\, r(\tau)\right]\)
Now an expectation under \(\bar{p}\) — estimable from old trajectories!

11.2 The Importance Weight

The ratio \(p_\theta(\tau) / \bar{p}(\tau)\) simplifies beautifully using the trajectory factorization:

Importance Weight — environment terms cancel
\[ \frac{p_\theta(\tau)}{\bar{p}(\tau)} = \frac{p(s_1)\prod_t \pi_\theta(a_t|s_t)\, p(s_{t+1}|s_t,a_t)}{p(s_1)\prod_t \bar{\pi}(a_t|s_t)\, p(s_{t+1}|s_t,a_t)} = \frac{\prod_{t=1}^T \pi_\theta(a_t|s_t)}{\prod_{t=1}^T \bar{\pi}(a_t|s_t)} \]

Again, the environment terms \(p(s_1)\) and \(p(s_{t+1}|s_t,a_t)\) cancel — they are the same under both policies. Only the policy action probabilities differ.

11.3 Off-Policy Gradient

Off-Policy Policy Gradient
\[ \nabla_\theta J(\theta) = \mathbb{E}_{\tau \sim \bar{p}(\tau)}\!\left[\left(\prod_{t=1}^T \frac{\pi_\theta(a_t|s_t)}{\bar{\pi}(a_t|s_t)}\right)\!\left(\sum_t \nabla_\theta \log \pi_\theta(a_t|s_t)\right)\!\left(\sum_t r(s_t,a_t)\right)\right] \]
The Exponential Weight Problem
The importance weight is a product of \(T\) ratios. For long trajectories, this product can be astronomically large or vanishingly small — exponential in \(T\). This is the fundamental challenge of off-policy learning. PPO addresses this by restricting how far \(\pi_\theta\) can deviate from \(\bar{\pi}\), keeping the importance weights well-behaved.

Applying causality and the baseline to the off-policy gradient brings us to the per-timestep formulation that PPO uses, where the importance weight at timestep \(t\) is just the ratio of action probabilities at that step: \(\pi_\theta(a_t|s_t) / \bar{\pi}(a_t|s_t)\).

12

Actor-Critic: Online TD-Based Policy Gradient

REINFORCE has two serious remaining problems. Actor-Critic fixes them simultaneously.

12.1 Problems with REINFORCE

Problem Root Cause Consequence
High variance Return \(G_t = \sum_{t'=t}^T r_{t'}\) sums up to \(T-t\) random variables Gradient estimates are noisy; requires many samples to converge
Episode-based updates Must observe the entire episode before computing \(G_t\) Cannot update online; inefficient for long episodes or continuing tasks

12.2 The Actor-Critic Architecture

Actor-Critic introduces a second learner — the Critic — that estimates the value function \(V^\omega(s)\). This allows us to replace the noisy Monte Carlo return with a lower-variance bootstrap estimate.

Actor: \(\pi_\theta(a|s)\)

A policy network parameterized by \(\theta\). Takes state \(s\), outputs action distribution. Learns via gradient ascent weighted by advantage. The "decision maker."

Update rule: \(\theta \leftarrow \theta + \alpha_\pi \cdot \delta_t \cdot \nabla_\theta \log \pi_\theta(a_t|s_t)\)

Critic: \(V_\omega(s)\)

A value network parameterized by \(\omega\). Takes state \(s\), outputs a scalar — the expected return from \(s\). Learns by minimizing TD error squared. The "evaluator."

Update rule: \(\omega \leftarrow \omega - \alpha_V \cdot \delta_t \cdot \nabla_\omega V_\omega(s_t)\)

12.3 The TD Error as Advantage Estimate

At each timestep, after observing the transition \((s_t, a_t, r_t, s_{t+1})\), we compute the TD error:

TD Error = 1-step Advantage Estimate
\[ \delta_t = r_t + \gamma V_\omega(s_{t+1}) - V_\omega(s_t) \]

Why does this equal the advantage? Apply the Bellman equation \(Q^\pi(s_t,a_t) = r_t + \gamma V^\pi(s_{t+1})\):

Derivation
\[ A^\pi(s_t,a_t) = Q^\pi(s_t,a_t) - V^\pi(s_t) = \underbrace{r_t + \gamma V^\pi(s_{t+1})}_{\text{Bellman equation for }Q^\pi} - V^\pi(s_t) = \delta_t \]
\(V_\omega(s_t)\)
"Before this step, I expected to earn this much total reward from \(s_t\)."
\(r_t + \gamma V_\omega(s_{t+1})\)
"I actually got \(r_t\) now, plus the discounted value of where I ended up."
\(\delta_t > 0\)
Outcome was better than expected → reinforce action \(a_t\).
\(\delta_t < 0\)
Outcome was worse than expected → suppress action \(a_t\).
\(\delta_t = 0\)
Exactly as expected → no gradient signal, no update needed.
The Neuroscience Connection
This TD error signal is strikingly similar to dopamine prediction error in the brain. Dopamine neurons fire more strongly than baseline when reward is better than predicted, less strongly when worse, and at baseline when exactly as predicted. The brain appears to implement temporal difference learning — one of the deepest connections between neuroscience and RL theory.

12.4 Bias-Variance Tradeoff

The TD error has dramatically lower variance than the Monte Carlo return, but introduces bias — because \(V_\omega\) is imperfect, especially early in training. This is the fundamental tradeoff:

Method Bias Variance When to use
REINFORCE (\(G_t\)) Zero Very High Short episodes, simple environments
1-step TD (\(\delta_t\)) Some Low Online learning, long episodes
n-step TD (\(R_t^{(n)}\)) Moderate Moderate Intermediate (A2C default n=5)
GAE (\(\hat{A}_t^\text{GAE}\)) Tunable (\(\lambda\)) Tunable (\(\lambda\)) PPO (λ=0.95 is standard)
13

A2C: Advantage Actor-Critic & Entropy

13.1 n-step Returns

A2C collects \(n\) steps before updating. The n-step return provides less bias than 1-step TD while staying lower variance than Monte Carlo:

n-step Return
\[ R_t^{(n)} = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots + \gamma^{n-1} r_{t+n-1} + \gamma^n V_\omega(s_{t+n}) \]

The bias from bootstrapping is discounted by \(\gamma^n\): as \(n\) increases, the imperfect critic's estimate counts for less, reducing bias. Variance increases because more real random rewards are included. Typical A2C uses \(n = 5\).

13.2 Entropy Bonus: Preventing Collapse

A critical practical problem: without regularization, policies often collapse early in training — one action gets probability near 1, all others near 0. If this action is suboptimal, the agent can never recover.

Shannon entropy of the policy distribution measures randomness:

Policy Entropy
\[ H(\pi(\cdot|s)) = -\sum_a \pi(a|s) \log \pi(a|s) \]

Maximum entropy: uniform distribution (maximum exploration). Zero entropy: deterministic policy (no exploration).

We add entropy as a bonus to the objective, encouraging the policy to remain uncertain and exploratory:

A2C Full Joint Loss (minimize)
\[ \mathcal{L}(\theta, \omega) = \underbrace{-\frac{1}{T}\sum_t \log \pi_\theta(a_t|s_t)\hat{A}_t}_{\text{policy loss (actor)}} + \underbrace{c_1 \frac{1}{T}\sum_t (V_\omega(s_t) - R_t^{(n)})^2}_{\text{value loss (critic)}} - \underbrace{c_2 \frac{1}{T}\sum_t H(\pi_\theta(\cdot|s_t))}_{\text{entropy bonus}} \]
Policy loss
Negative because we minimize this loss function but want to maximize \(J(\theta)\). Gradient ascent on \(J\) = gradient descent on \(-J\).
Value loss
MSE between critic's prediction \(V_\omega(s_t)\) and the n-step return target. Trains the critic to be accurate.
Entropy bonus
Negative sign because we minimize the loss but maximize entropy. Higher entropy = more exploration = better. \(c_2 \approx 0.01\).
\(c_1 \approx 0.5\)
Value loss coefficient. Critic learns at half the effective rate to prevent it from dominating the actor's gradient.
14

GAE & PPO

14.1 Generalized Advantage Estimation (GAE)

PPO uses GAE — a principled way to continuously interpolate between 1-step TD (low variance, high bias) and Monte Carlo (high variance, zero bias) using a single parameter \(\lambda\).

Start with the 1-step TD error: \(\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)\).

Note that the n-step return minus baseline equals a sum of discounted TD errors:

Proof: n-step advantage = sum of TD errors
\[\begin{aligned} R_t^{(n)} - V(s_t) &= r_t + \gamma r_{t+1} + \cdots + \gamma^{n-1}r_{t+n-1} + \gamma^n V(s_{t+n}) - V(s_t) \\ &= [r_t + \gamma V(s_{t+1}) - V(s_t)] \\ &\quad + \gamma[r_{t+1} + \gamma V(s_{t+2}) - V(s_{t+1})] \\ &\quad + \cdots \\ &= \sum_{k=0}^{n-1} \gamma^k \delta_{t+k} \end{aligned}\]

GAE takes a \(\lambda\)-weighted average of all n-step advantages:

GAE definition
\[ \hat{A}_t^{\text{GAE}(\lambda)} = (1-\lambda)\sum_{n=1}^\infty \lambda^{n-1} \hat{A}_t^{(n)} \]

This simplifies to a remarkable closed form:

GAE — closed form
\[ \hat{A}_t^{\text{GAE}(\gamma,\lambda)} = \sum_{k=0}^\infty (\gamma\lambda)^k \delta_{t+k} = \delta_t + \gamma\lambda\,\delta_{t+1} + (\gamma\lambda)^2\delta_{t+2} + \cdots \]
\(\lambda = 0\)
Only \(\delta_t\) survives: pure 1-step TD. \(\hat{A}_t = r_t + \gamma V(s_{t+1}) - V(s_t)\). High bias (V must be accurate), low variance.
\(\lambda = 1\)
Sum of all TD errors = Monte Carlo advantage \(G_t - V(s_t)\). Zero bias, high variance.
\(\lambda = 0.95\)
PPO default. Effective lookback \(\approx 1/(1-\lambda) = 20\) steps. Good balance in practice.

GAE is computable in \(O(T)\) by working backwards: \(\hat{A}_T = \delta_T\), then \(\hat{A}_t = \delta_t + \gamma\lambda\,\hat{A}_{t+1}\).

14.2 The Training Instability Problem

With the advantage in hand, A2C updates using importance sampling when running multiple gradient steps on the same batch. The ratio:

Probability ratio
\[ r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\text{ref}}(a_t|s_t)} \]

The unclipped surrogate objective \(L^{\text{PG}} = \mathbb{E}[r_t(\theta)\hat{A}_t]\) has a critical flaw: when \(\hat{A}_t > 0\), it incentivizes making \(r_t(\theta)\) as large as possible — pushing the new policy arbitrarily far from the old one. A catastrophically bad new policy generates bad data, which makes the next update even worse. Performance can collapse permanently.

14.3 PPO: The Clipped Surrogate Objective

PPO's solution: clip the ratio \(r_t(\theta)\) to stay within \([1-\varepsilon, 1+\varepsilon]\), where \(\varepsilon \approx 0.2\).

PPO Clipped Objective
\[ L^{\text{CLIP}}(\theta) = \mathbb{E}_t\!\left[\min\!\left(r_t(\theta)\hat{A}_t,\; \text{clip}(r_t(\theta), 1-\varepsilon, 1+\varepsilon)\cdot\hat{A}_t\right)\right] \]
Why min() is the right choice — case analysis

Case 1: \(\hat{A}_t > 0\) (action was good, want to reinforce it).

If \(r_t > 1+\varepsilon\): unclipped \(= r_t \hat{A}_t > (1+\varepsilon)\hat{A}_t\) = clipped. \(\min\) selects clipped — prevents over-reinforcing.

If \(r_t \leq 1+\varepsilon\): both equal, clip has no effect.

Case 2: \(\hat{A}_t < 0\) (action was bad, want to suppress it).

If \(r_t < 1-\varepsilon\): unclipped \(= r_t \hat{A}_t\) (less negative), clipped \(= (1-\varepsilon)\hat{A}_t\) (more negative). \(\min\) selects clipped — prevents over-penalizing.

Result: In both cases, the gradient can never benefit from moving \(r_t\) outside \([1-\varepsilon, 1+\varepsilon]\). The policy is implicitly constrained to a trust region around \(\pi_{\text{ref}}\). No second-order optimization required.

14.4 Complete PPO Objective

PPO Full Loss
\[ \mathcal{L}(\theta) = \mathbb{E}_t\!\left[L_t^{\text{CLIP}}(\theta) - c_1\underbrace{(V_\theta(s_t) - R_t)^2}_{\text{critic loss}} + c_2\underbrace{H(\pi_\theta(\cdot|s_t))}_{\text{entropy}}\right] \]

where \(R_t = \hat{A}_t + V(s_t)\) is the return target, and \(c_1 \approx 0.5\), \(c_2 \approx 0.01\).

Algorithm: PPO (Schulman et al. 2017)
1.
Collect \(T\) timesteps using current policy \(\pi_\theta\), storing \((s_t, a_t, r_t, s_{t+1})\).
2.
Compute TD errors \(\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)\).
3.
Compute GAE advantages: \(\hat{A}_t = \sum_k (\gamma\lambda)^k \delta_{t+k}\) (backward pass).
4.
Compute return targets: \(R_t = \hat{A}_t + V(s_t)\).
5.
For \(K\) epochs, mini-batch SGD on the full PPO loss \(\mathcal{L}(\theta)\) with clipped ratio. The reference policy \(\pi_{\text{ref}}\) is the policy that collected the data (fixed for all \(K\) epochs).
6.
Return to step 1 with updated \(\theta\).

PPO is currently the dominant RL algorithm in production: it powers RLHF training for ChatGPT and Claude alignment, OpenAI Five, robotics locomotion, and most large-scale RL applications. Its success comes from combining sample efficiency (multiple gradient steps per batch) with training stability (clipping) and simplicity (first-order optimization only).

TRPO & SAC: Advanced Methods

15.1 TRPO: The Theoretical Foundation

TRPO (Trust Region Policy Optimization, Schulman et al. 2015) is PPO's theoretical predecessor. It proved that you can guarantee monotonic improvement by solving a constrained optimization at each step:

TRPO Constrained Problem
\[ \max_\theta\; L^{\text{PG}}(\theta) \quad \text{subject to} \quad \overline{D}_{\text{KL}}(\pi_\theta \| \pi_{\text{old}}) \leq \delta \]
Monotonic Improvement Theorem (sketch)

Define the surrogate: \(\hat{L}^{\pi}(\pi') = J(\pi) + \mathbb{E}_{s \sim d^\pi}\mathbb{E}_{a \sim \pi'}[A^\pi(s,a)]\)

Then: \(J(\pi') \geq \hat{L}^\pi(\pi') - C \cdot \max_s D_{\text{KL}}(\pi'\|\pi)\)

where \(C = \frac{4\gamma\varepsilon}{(1-\gamma)^2}\) and \(\varepsilon = \max_{s,a}|A^\pi(s,a)|\).

If KL is bounded by \(\delta\), then \(J(\pi') \geq \hat{L}^\pi(\pi') - C\delta\). Maximizing \(\hat{L}^\pi(\pi')\) within the trust region guarantees \(J(\pi') \geq J(\pi)\). Every update makes things at least as good.

Monotonic Improvement Guarantee
\[ J(\pi_{\text{new}}) \geq J(\pi_{\text{old}}) - \frac{4\gamma\varepsilon}{(1-\gamma)^2} \overline{D}_{\text{KL}}(\pi_{\text{new}}\|\pi_{\text{old}}) \]

TRPO solves the constrained problem with conjugate gradient descent and backtracking line search. This requires computing the Fisher Information Matrix — expensive in practice. PPO achieves the same practical behavior with clipping, at a fraction of the computational cost.

15.2 SAC: Maximum Entropy RL

Soft Actor-Critic (SAC, Haarnoja et al. 2018) takes a fundamentally different approach: instead of just maximizing expected return, it maximizes expected return plus policy entropy simultaneously.

SAC Maximum Entropy Objective
\[ J(\pi) = \mathbb{E}_\tau\!\left[\sum_t \gamma^t \bigl(r(s_t, a_t) + \alpha H(\pi(\cdot|s_t))\bigr)\right] \]

The temperature \(\alpha\) controls the trade-off between reward maximization and entropy maximization. Crucially, the entropy term is part of the Bellman equation itself — the soft Bellman backup:

SAC Bellman Target (twin-Q)
\[ y = r + \gamma\!\left(\min_{j=1,2} Q_{\bar\theta_j}(s', a') - \alpha \log \pi(a'|s')\right) \]
\(\min_{j=1,2} Q\)
Twin Q-networks: two critics, take the minimum. This prevents overestimation bias — a known failure mode where a single Q network develops over-optimistic values.
\(-\alpha \log \pi(a'|s')\)
Entropy term built into the target. The soft value of a state naturally accounts for exploration value — states with many good actions are valued higher.
\(\alpha\)
Temperature — can be learned automatically by solving a dual optimization that targets a desired entropy level. This eliminates a critical hyperparameter.
Property PPO SAC
Policy type On-policy Off-policy (replay buffer)
Action space Discrete or continuous Primarily continuous
Sample efficiency Moderate High (5–10× better)
Stability High High
Best use case Discrete actions, fast simulation, RLHF Continuous control, real robots (expensive data)
Key innovation Clipped surrogate + GAE Maximum entropy + twin Q
Choosing between PPO and SAC
Use PPO when: you have fast simulation, discrete or mixed action spaces, or are doing RLHF/LLM alignment. Use SAC when: data collection is expensive (real robot hardware), the action space is continuous (robot joint torques), and you need maximum sample efficiency. Both algorithms are battle-tested in production systems.
