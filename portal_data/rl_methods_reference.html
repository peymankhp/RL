RL DP MC TD — Core Formulas

Complete Formula Reference — from the docx study material

1
Markov Decision Process (MDP)
Transition Dynamics
\[ P(s', r \mid s, a) = \Pr\{S_t = s',\; R_t = r \mid S_{t-1} = s,\; A_{t-1} = a\} \]
Markov Property
\[ P(S_{t+1} \mid S_t, S_{t-1}, \ldots, S_0) = P(S_{t+1} \mid S_t) \]
sThe current state.
aThe action taken by the agent.
s'The next state the agent transitions into.
rThe immediate reward received after taking action a.
PThe transition probability — the "dynamics" of the environment.
Key Points
  • Markov Property: the future depends only on the current state, not on any history before it.
  • Mathematical foundation of all RL problems — every RL algorithm implicitly assumes an MDP structure.
  • The agent has no control over P; it can only influence which state-action pairs it visits.
2
Bellman Equations
Bellman Expectation Equation
\[ V^{\pi}(s) = \sum_{a} \pi(a \mid s) \sum_{s', r} P(s', r \mid s, a) \left[ r + \gamma V^{\pi}(s') \right] \]
Bellman Optimality Equation
\[ V(s) = \max_a \sum_{s', r} p(s', r \mid s, a) \left[ r + \gamma V(s') \right] \]
V(s)The value of the current state.
γDiscount factor (0–1): how much future rewards matter vs. immediate ones.
π(a|s)Policy: probability of taking action a given state s.
maxaChoose the action yielding the highest possible total value.
Key Points
  • Expectation form: averages over the policy π(a|s) — evaluates the value under a specific policy.
  • Optimality form: takes the max over actions — describes the value under the best possible policy.
  • Recursive relationship: value of a state = best immediate reward + discounted value of the next state.
  • γ = 0 → fully short-sighted; γ → 1 → values all future rewards equally.
3
Dynamic Programming (DP)
Policy Evaluation Update
\[ V_{k+1}(s) = \sum_a \pi(a \mid s) \sum_{s', r} p(s', r \mid s, a) \left[ r + \gamma V_k(s') \right] \]
Vk+1The updated value estimate in the next iteration.
VkThe old value estimate from the previous iteration.
Key Points
  • Requires a complete model of the environment — transition probabilities P must be fully known.
  • Alternates between Policy Evaluation (computing V for fixed π) and Policy Improvement (updating π greedily w.r.t. V).
  • Computationally expensive for large state spaces — suffers from the curse of dimensionality.
4
Expected Return & Value Functions
Return (Discounted Sum of Rewards)
\[ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \]
State-Value Function
\[ v_{\pi}(s) = \mathbb{E}_{\pi}\!\left[ G_t \mid S_t = s \right] = \mathbb{E}_{\pi}\!\left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t = s \right] \]
Action-Value Function
\[ q_{\pi}(s, a) = \mathbb{E}_{\pi}\!\left[ G_t \mid S_t = s, A_t = a \right] = \mathbb{E}_{\pi}\!\left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t = s, A_t = a \right] \]
GtTotal discounted return from time step t onwards.
vπ(s)Expected return starting from state s following policy π.
qπ(s,a)Expected return after taking action a in state s, then following π.
γDiscount factor: controls the trade-off between immediate and future rewards.
Key Points
  • vπ(s) answers: "how good is it to be in state s?" — averaging over all actions the policy might take.
  • qπ(s,a) answers: "how good is it to take action a in state s?" — more informative for decision making.
  • The relationship: \( v_{\pi}(s) = \sum_a \pi(a|s)\, q_{\pi}(s,a) \) — the state-value is the policy-weighted average of action-values.
5
Monte Carlo (MC) Methods
MC Return
\[ G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \]
State Value (MC Estimate)
\[ v_{\pi}(s) = \text{average of returns from state } s \]
Action Value (MC Estimate)
\[ q_{\pi}(s, a) = \text{average return after taking action } a \text{ in state } s \]
GtActual total return (sum of all rewards) from time t until episode end.
αLearning rate: how much new data overwrites old belief.
Key Points
  • Model-free: learns purely from experience — does not require knowledge of transition dynamics \( P(s'|s,a) \).
  • Only applies to episodic tasks — waits until the full episode ends to compute Gt.
  • High variance, zero bias — returns are real unbiased samples, but fluctuate significantly between episodes.
6
Temporal Difference (TD) Learning
TD Prediction (Approximation)
\[ V(s) \approx v_{\pi}(s) \]
TD(0) Update Rule
\[ V(S_t) \leftarrow V(S_t) + \alpha \left[ R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \right] \]
TD Error
\[ \delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \]
Rt+1+γV(St+1)TD Target: our new, slightly more accurate estimate of the value.
δtTD Error: the surprise felt when seeing the actual reward and next state.
V(St+1)The estimated value of the next state just landed in.
αLearning rate: controls how much to adjust the current estimate.
Key Points
  • Best of both worlds: model-free like MC, but learns online after every single step — no need to wait for episode end.
  • Bootstraps: updates estimates based on other estimates — introduces some bias but significantly reduces variance vs. MC.
  • TD(0) is the simplest form; TD(λ) generalizes across a spectrum from pure TD (λ=0) to pure MC (λ=1).
7
TD Error Decomposition — Full Proof
📐 Proof: Gt − V(St) = Σ γk−t δk
Goal: Show that the MC error decomposes into a sum of TD errors: \[ G_t - V(S_t) = \sum_{k=t}^{T-1} \gamma^{k-t} \delta_k \] where \( \delta_k = R_{k+1} + \gamma V(S_{k+1}) - V(S_k) \).
Step 1 — Expand the return recursively: \[ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots \] \[ G_t = R_{t+1} + \gamma G_{t+1} \]
Step 2 — Subtract V(St): \[ G_t - V(S_t) = R_{t+1} + \gamma G_{t+1} - V(S_t) \]
Step 3 — Add and subtract γV(St+1): \[ = R_{t+1} + \gamma V(S_{t+1}) - V(S_t) + \gamma \left( G_{t+1} - V(S_{t+1}) \right) \]
Step 4 — Recognise the TD error: \[ \delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \] \[ G_t - V(S_t) = \delta_t + \gamma \left( G_{t+1} - V(S_{t+1}) \right) \]
Step 5 — Unroll recursively: \[ = \delta_t + \gamma \delta_{t+1} + \gamma^2 \left( G_{t+2} - V(S_{t+2}) \right) \] \[ = \delta_t + \gamma \delta_{t+1} + \gamma^2 \delta_{t+2} + \cdots \]
Final Result:
MC Error = Discounted Sum of TD Errors
\[ \boxed{G_t - V(S_t) = \sum_{k=t}^{T-1} \gamma^{k-t} \delta_k} \]
Why This Matters
  • Bridges MC and TD: the total Monte Carlo error is simply the discounted sum of all one-step TD errors — they are two views of the same quantity.
  • Foundation of TD(λ): this decomposition is the mathematical basis for eligibility traces and the λ-return.
  • GAE (PPO): Generalized Advantage Estimation uses a weighted version of this decomposition to balance bias and variance in policy gradient methods.
